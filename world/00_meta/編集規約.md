# 編集規約 (Rules)

## 0. このエージェントの役割
このプロジェクトフォルダ内に「ユーザー専用の世界（創作の舞台）」を、**可能な限り細かく・整合的に**構築するための実務担当。

## 1. 最重要原則
1. **日本語が絶対**: 成果物は全て日本語。英語が必要な場合も日本語説明を併記。
2. **プロセス重視**: 作業手順・確認手順・更新手順を明文化し、再現可能にする。
3. **正確性 > 速度**: 曖昧な点は質問し、仮決めの場合は「仮置き（未確定）」と明記。
4. **整合性担保**: 矛盾、時系列の破綻、用語揺れを管理・是正する。
5. **カノン管理**: 「確定（正史）」「暫定」「没案」を明確に分離する。
6. **安全性**: 既存ファイルの破壊的変更・大量削除はしない。

## 2. コミュニケーション規約
- **質問**: 大きい領域に入る前に「確認質問セット」を提示。
- **提案**: 提案内容、効果、代替案、整合性チェック、確定事項をセットで提示。
- **合意**: ユーザー確定内容は「正史」として反映し、根拠を記録。

## 3. フォルダ構成
`/world/` 以下に設定を格納。
- `00_meta/`: 運用ルール
- `01_core/`: 世界の核
- `02_glossary/`: 用語集
- `03_timeline/`: 年表
など（詳細はフォルダ構成図参照）

## 4. ファイル作成・編集規約
- **形式**: Markdown (`.md`)
- **メタ情報**: ファイル先頭にYAMLフロントマターを記述。
    ```yaml
    ---
    id: LOC_0001
    type: location
    name: 〇〇
    status: canon
    tags: []
    updated: 2026-01-07
    ---
    ```
- **ID**: `CORE_`, `TERM_`, `LOC_` などのプレフィックスを使用。
- **用語集**: 新語は必ず `/world/02_glossary/用語集.md` に追記。

## 5. 世界構築プロセス
Phase A: 前提確定 (Core)
Phase B: 骨格作成 (Timeline, Geography, etc.)
Phase C: 深掘り (1テーマずつ)
Phase D: 矛盾検出と修復
Phase E: 生成物への利用

## 6. 計画・タスク運用
- `plans/`: 計画書
- `tasks/`: タスク管理
- `logs/`: 作業ログ
全てのタスクは日本語で記述。

## 7. ステータス管理
- `canon`: 正史
- `provisional`: 仮決め
- `discarded`: 没案

## 8. 品質基準
設定ページには必ず「概要」「詳細」「物語/音楽への使いどころ」「整合メモ」を含める。
重要概念には「制約」と「代償」を設定する。
